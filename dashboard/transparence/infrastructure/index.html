Infrastructure Overview

Vue publique et volontairement partielle de l'infrastructure Ocean Sentinel. Cette page explique les grands composants, les garanties de transparence et les limites publiables, sans exposer d'éléments d'administration, de revue opérationnelle ou de configuration confidentielle.

Version publique volontairement partielle. Les détails d'exploitation, inventaires techniques complets, règles de filtrage, identifiants d'administration et procédures de vérification ne sont pas publiés.

Hébergement

Ocean Sentinel est servi depuis un VPS Linux durci, avec séparation entre contenu statique, API applicative et données publiées.

Public summary

Publication

Les pages publiques et fichiers JSON de statut sont publiés de façon statique lorsque c'est possible, afin de limiter la surface d'exposition.

Static-first

Décisionnel

Les mesures actuellement affichées sont historiques (EXPIRED). Le système reste en shadow mode et ne déclenche aucune alerte opérationnelle.

Shadow mode

Architecture publique

  • Un frontal web sert le site public et les pages de transparence.
  • Une API applicative expose des endpoints documentés pour les statuts publics.
  • Les fichiers JSON publics sont séparés des données internes et des journaux d'exploitation.
  • La supervision existe, mais ses interfaces et détails opérationnels ne sont pas publics.

Transparence publiée

  • Source et statut des mesures sont affichés dans le dashboard.
  • Fraîcheur, décisionnel, limites et incertitude sont rendus visibles.
  • SCENARIO ≠ MEASURED : les simulations IA ne sont pas des mesures.
  • EXPIRED implique une absence d'alerte opérationnelle.

Ce qui n'est pas publié

Pour préserver la sécurité du service, cette page n'expose pas les inventaires détaillés, les accès d'administration, les chemins de configuration, les procédures de vérification, les paramètres de durcissement, les journaux, les identifiants ou les procédures confidentielles.
